# Overview（产品概述）结构标准

## 必需章节

| 章节 | 要求 |
|------|------|
| **产品简介** | 简要介绍产品是什么，解决什么问题，核心价值主张 |
| **核心功能** | 列出产品的主要功能特性，通常以列表形式呈现 |
| **应用场景** | 说明产品的典型应用场景，帮助用户理解何时使用该产品 |
| **产品架构** | 用图示说明产品的核心架构、主要组件及其关系 |
| **计费说明** | 说明计费模式（按分钟/按路数/免费额度等），引导至计费文档 |

## 可选章节

| 章节 | 要求 |
|------|------|
| **产品优势** | 对比竞品或同类方案的优势（性能、稳定性、功能等） |
| **版本说明** | 说明当前 SDK 版本对应的文档版本 |

## 检查要点

- 产品简介是否在一句话内说清楚产品是什么
- 是否有清晰的架构图
- 应用场景是否贴近实际业务
- 计费说明是否准确引导

## 输出格式

```markdown
## Overview 结构检查结果

### ✅ 结构完整
（所有必需章节都存在时）

### ❌ 缺失章节

1. **[章节名称]** - 缺失
   建议：[具体的改进建议]

### ⚠️ 章节内容不足

1. **[章节名称]**
   - 当前内容：[描述当前状态]
   - 建议：[具体的改进建议]
```
