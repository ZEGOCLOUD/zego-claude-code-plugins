# ReleaseNote（发布日志）结构标准

## 必需内容

| 内容 | 要求 |
|------|------|
| **版本号** | 清晰标注 SDK 版本号（如 v2.24.0） |
| **发布日期** | 标注发布日期 |
| **更新内容** | 按类型分类说明：新功能、功能优化、问题修复 |
| **升级指引** | 说明从旧版本升级的注意事项、API 变更、废弃接口等 |

## 格式要求

- 使用表格或折叠面板组织内容，便于快速浏览
- 新功能优先，其次是优化，最后是修复
- 问题修复应包含问题编号或问题描述
- 重要变更需突出显示（如 Breaking Changes）

## 检查要点

- 版本号格式是否统一
- 发布日期是否完整
- 是否有升级指引
- 敬语使用是否规范（使用"您"而非"你"）

## 输出格式

```markdown
## ReleaseNote 结构检查结果

### ✅ 结构完整
（所有必需内容都存在时）

### ❌ 缺失内容

1. **[内容名称]** - 缺失
   建议：[具体的改进建议]

### ⚠️ 内容不足

1. **[内容名称]**
   - 当前内容：[描述当前状态]
   - 建议：[具体的改进建议]
```
