# RunDemo（跑通示例代码）结构标准

## 必需章节

| 章节 | 要求 |
|------|------|
| **简介** | 说明该示例代码演示了哪些功能，最好有配图或 GIF |
| **准备环境** | 说明运行该示例代码的必要开发环境要求（IDE + 设备版本要求）。需理解整篇文档内容后倒推得出结论 |
| **前提条件** | 说明运行该示例代码的前提条件（如需要先在控制台创建项目，并申请有效的 AppID 和 AppSign）。需理解整篇文档内容后倒推得出结论 |
| **获取示例源码** | 提供本地下载链接或 GitHub 地址 |
| **示例源码目录结构** | 以目录树形式，说明示例源码核心目录下内容的作用 |
| **运行示例代码** | 说明运行示例代码的详细步骤，使用 Steps 组件，重点突出关键配置项（权限配置、关键 KEY 配置等） |
| **查看更多** | 引导用户查看快速开始、实现功能、集成 SDK 等相关文档 |

## 可选章节

| 章节 | 要求 |
|------|------|
| **概念解释** | 简要说明本产品或者 ZEGO 其他产品/平台概念的含义。尽可能添加含有更详细概念介绍的链接 |

## 检查要点

- 简介是否有配图
- 准备环境和前提条件是否区分清楚
- 运行步骤是否使用 Steps 组件
- 关键配置项是否突出显示

## 输出格式

```markdown
## RunDemo 结构检查结果

### ✅ 结构完整
（所有必需章节都存在时）

### ❌ 缺失章节

1. **[章节名称]** - 缺失
   建议：[具体的改进建议]

### ⚠️ 章节内容不足

1. **[章节名称]**
   - 当前内容：[描述当前状态]
   - 建议：[具体的改进建议]
```
