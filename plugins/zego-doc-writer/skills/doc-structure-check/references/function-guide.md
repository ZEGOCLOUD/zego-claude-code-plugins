# FunctionGuide（功能指南）结构标准

## 必需章节

| 章节 | 要求 |
|------|------|
| **功能简介** | 说明该功能是什么，解决什么问题 |
| **使用场景** | 列举该功能的典型使用场景 |
| **实现原理** | 简要说明该功能的实现原理或工作流程（可配图） |
| **使用前提** | 说明使用该功能的前置条件（如需要先开通某服务、完成某配置） |
| **实现步骤** | 分步骤说明实现该功能的详细流程 |
| **注意事项** | 说明使用该功能时需要注意的事项、限制、坑点 |

## 可选章节

| 章节 | 要求 |
|------|------|
| **API 说明** | 引用或列出相关的 API 接口 |
| **最佳实践** | 提供该功能使用的最佳实践建议 |

## 检查要点

- 功能描述是否清晰
- 是否有使用场景示例
- 代码示例是否完整
- 注意事项是否覆盖常见坑点

## 输出格式

```markdown
## FunctionGuide 结构检查结果

### ✅ 结构完整
（所有必需章节都存在时）

### ❌ 缺失章节

1. **[章节名称]** - 缺失
   建议：[具体的改进建议]

### ⚠️ 章节内容不足

1. **[章节名称]**
   - 当前内容：[描述当前状态]
   - 建议：[具体的改进建议]
```
