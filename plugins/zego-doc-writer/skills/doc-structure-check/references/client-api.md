# ClientAPI（客户端 API）结构标准

## 必需内容

| 内容 | 要求 |
|------|------|
| **API 概述** | 说明该 API 的作用 |
| **函数签名** | 完整的函数签名，包含参数类型和返回值类型 |
| **参数说明** | 每个参数的详细说明，包含类型、是否必填、默认值、取值范围 |
| **返回值说明** | 返回值的详细说明 |
| **回调说明** | 如有回调，说明回调参数含义 |
| **代码示例** | 提供完整的调用示例 |
| **注意事项** | 说明使用该 API 需要注意的事项 |

## 检查要点

- 函数签名是否与实际代码一致
- 参数说明是否完整
- 是否有代码示例
- 错误码是否说明

## 输出格式

```markdown
## ClientAPI 结构检查结果

### ✅ 结构完整
（所有必需内容都存在时）

### ❌ 缺失内容

1. **[内容名称]** - 缺失
   建议：[具体的改进建议]

### ⚠️ 内容不足

1. **[内容名称]**
   - 当前内容：[描述当前状态]
   - 建议：[具体的改进建议]
```
