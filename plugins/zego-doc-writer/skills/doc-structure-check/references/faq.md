# FAQ（常见问题）结构标准

## 必需内容

| 内容 | 要求 |
|------|------|
| **问题** | 以问题形式描述用户可能遇到的疑问 |
| **答案** | 清晰、准确的解答 |
| **相关链接** | 引用相关文档链接，便于深入了解 |

## 组织方式

- 按主题分类（如集成问题、功能问题、计费问题等）
- 或按频率排序（最常见的问题在前）

## 检查要点

- 问题是否真实反映用户疑惑
- 答案是否准确、可操作
- 是否有相关文档链接
- 是否保持更新（过时问题需移除或更新）

## 输出格式

```markdown
## FAQ 结构检查结果

### ✅ 结构完整
（所有必需内容都存在时）

### ❌ 缺失内容

1. **[内容名称]** - 缺失
   建议：[具体的改进建议]

### ⚠️ 内容不足

1. **[问题标题]**
   - 当前内容：[描述当前状态]
   - 建议：[具体的改进建议]
```
