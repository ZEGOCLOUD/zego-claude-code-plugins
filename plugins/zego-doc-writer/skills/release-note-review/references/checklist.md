# 发布日志审核清单

## 1. 必填内容检查

### 1.1 版本号
- [ ] 版本号是否存在
- [ ] 版本号格式是否正确（如：1.2.0、2.0.1）

### 1.2 发布日期
- [ ] 发布日期是否存在
- [ ] 日期格式是否合理

### 1.3 变更内容
- [ ] 变更内容是否存在
- [ ] 至少包含以下一项：
  - [ ] 新增功能
  - [ ] 改进优化
  - [ ] 废弃删除
  - [ ] 问题修复

## 2. 格式一致性检查

### 2.1 表现形式
- [ ] 遵循现有发布日志的表现形式
  - 表格形式
  - 折叠面板形式（`<Accordion>`）
  - 有序列表形式
  - 无序列表形式
- [ ] 除非用户明确要求，否则不改变原有格式

### 2.2 风格统一
- [ ] 新增内容与现有内容的风格保持一致
- [ ] 标题层级、列表样式、标点符号使用统一

## 3. 表达质量检查

### 3.1 敬语使用
- [ ] 指代用户时使用"您"而非"你"
- [ ] 整体语气专业、友好

### 3.2 新增功能
- [ ] 描述简洁明了，突出功能点
- [ ] 避免冗长重复
- [ ] 技术术语（功能名称、接口名称、技术名称等）配有链接说明

### 3.3 改进优化
- [ ] 描述简洁明了，重点突出优化点
- [ ] 避免冗长重复
- [ ] 技术术语配有链接说明

### 3.4 问题修复
- [ ] 避免对公司形象有负面影响的表述
- [ ] 使用委婉的表达方式
  - ❌ 修复计费逻辑错误导致多计费问题
  - ✅ 修复计费相关问题
- [ ] 描述纯技术性变更
  - ✅ 修复投屏到电视后漏回声的问题
- [ ] 避免明确说明对客户利益的损害

## 4. 表达流畅度检查

- [ ] 语句通顺，逻辑清晰
- [ ] 无错别字、语病
- [ ] 专业术语使用准确
- [ ] 标点符号使用正确
