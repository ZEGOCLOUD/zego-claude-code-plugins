# 发布日志审核与优化

## 审核流程

### 步骤 1：读取审核标准

首次执行时，读取 [checklist.md](references/checklist.md) 了解完整的审核标准和要求。

### 步骤 2：读取发布日志

打开相关的发布日志文件，只读取前 3 条发布日志，了解原来发布日志的格式。

### 步骤 3：读取示例参考【可选】

如需查看发布日志格式示例，读取 [examples.md](references/examples.md)。一般全新产品的发布日志或者用户主动要求改写发布日志格式时参考。

### 步骤 4：执行审核

对用户提供的发布日志进行审核，检查以下维度：

#### 必填内容检查
- 版本号是否存在且格式正确
- 发布日期是否存在
- 变更内容是否包含至少一项：新增功能、改进优化、废弃删除、问题修复

#### 格式一致性检查
- 是否遵循现有发布日志的表现形式（表格/折叠面板/列表）
- 除非用户明确要求，否则不改变原有格式
- 新增内容与现有内容的风格保持一致

#### 表达质量检查
- **敬语使用**：指代用户时使用"您"而非"你"
- **新增功能**：简洁明了，突出功能点，避免冗长重复，技术术语配有链接
- **改进优化**：简洁明了，重点突出优化点，避免冗长重复，技术术语配有链接
- **问题修复**：
  - 避免对公司形象有负面影响的表述（如："修复计费逻辑错误导致多计费问题" → "修复计费相关问题"【多计费明显会损害客户利益】）
  - 使用委婉表达
  - 描述纯技术性变更

### 步骤 4：反馈与改写

审核完成后：
1. **逐条列出**发现的问题。要给出问题序号方便用户指明哪些要修改。
2. **提供具体的优化建议**
3. **询问用户**是否接受建议

如果用户接受建议，按照原有格式风格改写发布日志。

## 语法参考

在编写日志内容时可能会存在使用一些组件和特殊语法的，这里提供一些参考。

### Accordion - 折叠面板

用于可折叠展开的内容块。

**属性：**
| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| title | `string` | - | 折叠面板标题 |
| defaultOpen | `"true" \| "false"` | `"false"` | 是否默认展开 |

**示例：**
```mdx
<Accordion title="点击展开详情" defaultOpen="false">
  这里是折叠的详细内容。
</Accordion>
```

### Callout - 提示框

用于显示不同类型的提示信息。

**快捷组件（推荐）：**
```mdx
<Note title="注意">这是一条注意事项</Note>
<Tip title="提示">这是一条提示信息</Tip>
<Warning title="警告">这是一条警告信息</Warning>
<Error title="错误">这是一条错误信息</Error>
```

### 表格增强语法

VSCode 插件提供了表格增强功能，支持列宽控制、对齐方式和单元格合并。

### 列宽设置

在表头单元格后添加 `-宽度` 设置列宽：

```markdown
| 名称-30% | 描述-50% | 备注-20% |
|----------|----------|----------|
| 内容1    | 内容2    | 内容3    |
```

支持百分比和像素值：
- 百分比：`标题-30%`
- 像素值：`标题-120px`

### 对齐方式

在表头单元格后添加 `-l`、`-c`、`-r` 设置对齐：

```markdown
| 左对齐-l | 居中-c | 右对齐-r |
|----------|--------|----------|
| 左       | 中     | 右       |
```

- `-l`：左对齐（left）
- `-c`：居中（center）
- `-r`：右对齐（right）

### 组合使用

列宽和对齐可以组合使用：

```markdown
| 名称-30%-l | 数值-20%-r | 描述-50%-c |
|------------|------------|------------|
| 苹果       | 100        | 水果       |
```

### 单元格合并

在需要合并的单元格中使用合并标记：

| 标记 | 说明 |
|------|------|
| `!mu` 或 `!m` | 向上合并（与上方单元格合并） |
| `!md` | 向下合并（与下方单元格合并） |
| `!ml` | 向左合并（与左侧单元格合并） |
| `!mr` | 向右合并（与右侧单元格合并） |

**纵向合并示例（向上合并）：**

```markdown
| 类别   | 项目   | 说明   |
|--------|--------|--------|
| 水果   | 苹果   | 红色   |
| !mu    | 香蕉   | 黄色   |
| !mu    | 橙子   | 橙色   |
| 蔬菜   | 白菜   | 绿色   |
```

渲染效果："水果" 单元格会纵向跨越 3 行。

**横向合并示例（向左合并）：**

```markdown
| 姓名   | 联系方式       | !ml    |
|--------|----------------|--------|
| 张三   | 电话           | 邮箱   |
```

渲染效果："联系方式" 表头会横向跨越 2 列。
