# 内容准确性及完整性检查

## 检查维度

站在完全没使用过 ZEGO 产品的用户角度，多提"为什么？"和"是什么？"。如果从整篇文档中无法找到答案，则认为内容不完整、不准确。

### 1. 敬语使用

检查指代用户时是否使用"您"而非"你"。

### 2. 数据、参数获取方式

检查是否说明了作用及获取方式。

**示例**：
- AppID 和 Token 的获取方式
- AppSign 的获取方式
- 服务器地址的获取方式

### 3. 技术概念/术语定义

检查是否简要说明了技术概念/术语的含义，并尽可能添加含有更详细介绍的站内链接。

**示例**：
- AppID、Token、AppSign
- RTC 房间、流
- 推流、拉流

### 4. 功能及接口链接

检查功能名称、接口名称、技术名称等是否配有对应链接。

**为功能/接口/技术术语添加链接的方法**：

1. **本产品功能及接口**：
   - 搜索本实例或本产品内所有实例的 sidebars.json 文件
   - 从 label 及 id 字段值判断是否与技术术语相关
   - 打开文件内容确认相关性
   - 使用 `python3 .docuo/scripts/config_helper.py <mdx_file_path> --url` 获取 URL

2. **其他产品技术概念/术语**：
   - 在整个仓库内搜索技术术语
   - 选择最相关的文档路径
   - 使用上述命令获取 URL

3. **锚点链接**（可细化到锚点时）：
   - 锚点格式：`#### 文档结构检查 <a id="document-structure-check" />`
   - 锚点只能是英文加连接符，不能是中文
   - ParamField 组件会自动生成锚点，参考 [ParamField Anchor Guide](references/param-field-anchor-guide.md)

### 5. 敏感性表述检查

禁止出现：
- 对公司有负面影响的描述
- 涉军涉政等敏感表述

## 输出格式

```markdown
## 内容准确性及完整性检查结果

### ✅ 检查通过
所有检查项均通过，未发现内容准确性或完整性问题。

### ❌ 发现的问题

1. **敬语使用**
   - 第 X 行：使用了"你"，应改为"您"

2. **数据/参数获取方式缺失**
   - 第 X 行：提到 AppID 但未说明获取方式

3. **技术概念/术语未定义或无链接**
   - 第 X 行：提到"Token"但未说明含义或添加链接

4. **功能/接口缺少链接**
   - 第 X 行：提到"createRoom"方法但未添加链接

5. **敏感性表述**
   - 第 X 行：[具体问题]
```
