# 文档结构检查

## 检查标准

站在完全没用过 ZEGO 产品的用户角度，审核文档结构是否清晰、完整，是否容易理解。

### 必需章节

| 章节 | 要求 |
|------|------|
| **简介** | 说明该示例代码演示了哪些功能，最好有配图 |
| **准备环境** | 说明运行该示例代码的必要开发环境要求（最小化要求，一般是 IDE + 开发设备版本要求）。需理解整篇文档内容后倒推得出结论 |
| **前提条件** | 说明运行该示例代码的前提条件（如需要先在控制台创建项目，并申请有效的 AppID 和 AppSign）。需理解整篇文档内容后倒推得出结论 |
| **获取示例源码** | 提供本地下载链接或 GitHub 地址 |
| **示例源码目录结构** | 以目录树形式，说明示例源码核心目录下内容的作用 |
| **运行示例代码** | 说明运行示例代码的详细步骤，使用 Steps 组件，重点突出关键配置项（权限配置、关键 KEY 配置等） |
| **查看更多** | 引导用户查看快速开始、实现功能、集成 SDK 等相关文档 |

### 可选章节

| 章节 | 要求 |
|------|------|
| **概念解释** | 简要说明本产品或者 ZEGO 其他产品/平台概念的含义。尽可能添加含有更详细概念介绍的链接 |

## 输出格式

```markdown
## 文档结构检查结果

### ✅ 结构完整
（所有必需章节都存在时）

### ❌ 缺失章节

1. **简介** - 缺失
   建议：添加示例功能简介说明，并配上截图

2. **前提条件** - 缺失
   建议：说明需要先在控制台创建项目并获取 AppID

### ⚠️ 章节内容不足

1. **准备环境**
   - 当前内容：仅说明需要 Xcode
   - 建议：补充 macOS 版本要求、iOS 设备版本要求

2. **运行示例代码**
   - 当前内容：步骤描述不够详细
   - 建议：使用 Steps 组件，并突出关键配置项
```
